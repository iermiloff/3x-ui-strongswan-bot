# 📑 White Label VPN Automation Platform (XRAY + IKEv2 StrongSwan)

Высокотехнологичное модульное решение (White Label) для автоматизации дистрибуции, коммерциализации и централизованного управления приватной VPN-инфраструктурой через Telegram-бота. 

Платформа спроектирована с полной изоляцией биллинга и бизнес-логики от сетевых шлюзов, что позволяет администратору разворачивать архитектуру в любых топологиях и масштабировать сеть без изменения исходного кода.

---

## ⚡️ Ключевые преимущества и возможности ядра

### 🌐 1. Полная нативная интеграция с 3x-ui (MHSanaei)
Архитектура системы полностью синхронизирована с API последних версий передовой панели **3x-ui от MHSanaei**. Бот в автоматическом режиме управляет инбаундами, на лету нарезает уникальные клиентские UUID, привязывает email-идентификаторы и кэширует публичные ключи Reality в ОЗУ.

### 🔀 2. Уникальный гибридный стек: 3x-ui + StrongSwan
Платформа реализует одновременную параллельную работу двух принципиально разных сетевых демонов на целевых нодах:
* **Ядро Xray Core** — отвечает за маскируемые Stealth-протоколы нового поколения для десктопов и тяжелого трафика.
* **Демон StrongSwan** — разворачивает нативный туннель промышленного стандарта безопасности, идеального для работы мобильных устройств и роутеров.

### 🔑 3. Автоматизированная выдача мультиключей (Накопительные тарифы)
Внедрена умная накопительная логика подписок, исключающая замусоривание серверов:
* **Тариф БАЗОВЫЙ:** Генерирует пользователю пакет из чистых протоколов **XRAY (VLESS Reality / Trojan Reality)** для работы через сторонние утилиты.
* **Тариф PREMIUM:** Автоматически включает в себя все доступы Базового тарифа + **нативно нарезает учетную запись IKEv2**. При покупке или апгрейде подписки система не плодит дубликаты портов, а бесшовно продлевает существующие XRAY-клиенты, параллельно работая с secrets-файлом Strongswan.

### 🛠 4. Абсолютная автоматизация и контроль для Администратора
Система полностью автономна и предоставляет администратору бескомпромиссный уровень QoL-удобства (Quality-of-Life):
* **Полный White Label:** Сетка цен, сроки подписок, текстовое брендирование (имя сервиса) и валюта приема платежей полностью изолированы в файле среды `.env`. Для изменения бизнес-модели или ребрендинга достаточно отредактировать текстовые параметры.
* **Ручной оверрайд:** Интегрирован полноценный административный пульт прямо в Telegram. Если у клиента завис фиатный перевод или транзакция в блокчейне, админ в три клика может вручную зачислить, продлить или аннулировать любой тариф по Telegram ID в обход платежного шлюза. Бот сам обновит базы данных и мгновенно перенарежет ключи на серверах.
* **Бесшовный шедулер:** Фоновый планировщик задач (`APScheduler`) круглосуточно мониторит СУБД. При истечении срока подписки бот за 0.001 секунды отключает порты в 3x-ui и комментирует строку пользователя по SSH в файле секретов StrongSwan. Текущие сессии других платящих клиентов при этом никогда не прерываются.

## 🛠 2. Технологический стек проекта

Платформа построена на базе современного асинхронного стека, что обеспечивает высокую скорость обработки запросов, устойчивость к нагрузкам и простоту контейнеризации.

*   **Application Layer (Back-end):** `Python 3.11` / `Aiogram 3.x` (полностью асинхронный фреймворк для Telegram-ботов с поддержкой хуков жизненного цикла и FSM-контекстов).
*   **Database & ORM:** `PostgreSQL` / `SQLAlchemy 2.0 (Async)` / `Alembic` (автоматическая миграция структур данных и строгая типизация связей).
*   **Automation Stores & Task Management:** `APScheduler` (высокоточный фоновый планировщик задач мониторинга подписок).
*   **Network Transport Layer:** `AsyncSSH` (асинхронное SSH-взаимодействие с удаленными нодами на базе безопасных протоколов).
*   **Routing Core:** `Xray Core` (модификация от MHSanaei) / `StrongSwan` (EAP-MSCHAPv2 / Linux Policy Based Routing таблиц ядра).
*   **DevOps:** `Docker` / `Docker-compose` (полная изоляция среды выполнения и зависимостей).

## 🚀 3. Быстрый старт и развертывание

Развертывание системы разделено на два этапа и может быть выполнено как на одном физическом сервере, так и на независимых машинах.

### Шаг 1. Подготовка Сетевой Ноды
На целевом сервере (рекомендуется `Ubuntu 22.04 / 24.04 LTS`) необходимо:
1. Перевести стандартный порт SSH на кастомный высокий порт (например, `52222`) для обхода гео-блокировок 22 порта от магистральных провайдеров.
2. Установить актуальную версию панели `3x-ui` от MHSanaei.
3. Развернуть демон `StrongSwan` и загрузить коммерческие SSL-сертификаты вашего домена.
4. Включить IPv4/IPv6 форвардинг пакетов и прописать правила `Policy Based Routing` в `/etc/rc.local`, чтобы нативный мобильный трафик IKEv2 принудительно заворачивался в GeoIP-интерфейс панели.

### Шаг 2. Запуск Управляющего Бота
На сервере биллинга выполните стандартные команды сборки:

```bash
# Клонирование репозитория платформы
git clone https://github.com/iermiloff/3x-ui-strongswan-bot/
cd 3x-ui-strongswan-bot/

# Создание и настройка конфигурации окружения
cp .env.example .env
nano .env

# Сборка и запуск всей экосистемы в фоне
docker-compose up -d --build
```
