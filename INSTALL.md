## 🧭 ЧАСТЬ 1. ПОДГОТОВКА СЕТЕВОЙ НОДЫ (XRAY + STRONGSWAN)

В качестве базовой операционной системы рекомендуется использовать чистый образ **Ubuntu 22.04 LTS** или **Ubuntu 24.04 LTS**. Все команды выполняются от имени пользователя `root`.

### Шаг 1.1. Маскировка управляющего шлюза (Смена порта SSH)
Для защиты от автоматического зарубежного брутфорса и обхода возможных ограничений хостинга переведем системный порт SSH со стандартного `22` на скрытый высокий порт (например, `52222`).

1. Откройте главный конфигурационный файл демона SSH:
   ```bash
   nano /etc/ssh/sshd_config
   ```
2. Найдите директиву `Port` (или добавьте её, если она закомментирована) и укажите кастомный порт:
   ```text
   Port 52222
   ```
3. В современных дистрибутивах Ubuntu запуском SSH управляет сокет-активатор `systemd`. Создадим конфигурационный файл переопределения сокета, иначе система продолжит удерживать порт `22`:
   ```bash
   mkdir -p /etc/systemd/system/ssh.socket.d/
   ```
4. Запишите правила сокета (явное указание слушать порт на IPv4 и IPv6 интерфейсах):
   ```bash
   cat <<EOF > /etc/systemd/system/ssh.socket.d/listen.conf
   [Socket]
   ListenStream=
   ListenStream=0.0.0.0:52222
   ListenStream=[::]:52222
   EOF
   ```
5. Перезапустите менеджер конфигураций ядра и обновите сетевые сокеты:
   ```bash
   systemctl daemon-reload
   systemctl restart ssh.socket
   systemctl restart ssh
   ```
6. Убедитесь, что сервер успешно пересел на новый порт:
   ```bash
   ss -tlnp | grep ssh
   ```
   *Вывод должен содержать строки вида `0.0.0.0:52222` и `[::]:52222`.*

### Шаг 1.3. Установка и конфигурация нативного шлюза StrongSwan (IKEv2)
1. Установите программный комплекс StrongSwan и дополнительные плагины шифрования:
   ```bash
   apt update && apt install strongswan strongswan-pki libcharon-extra-plugins -y
   ```
2. Разместите ваш SSL-сертификат (выпущенный для домена вашей ноды) строго по системным каталогам:
   * Приватный ключ домена (`.key` или `.pem` без пароля) -> `/etc/ipsec.d/private/server-key.pem`
   * Файл корневого сертификата удостоверяющего центра (CA) -> `/etc/ipsec.d/cacerts/ca-cert.pem`
   * Файл самого сертификата домена (включая промежуточные звенья) -> `/etc/ipsec.d/certs/server-cert.pem`

3. Настройте конфигурацию туннеля. Полностью очистите файл `/etc/ipsec.conf` и вставьте следующую структуру:
   ```text
   config setup
       uniqueids=yes

   conn ikev2-vpn
       keyexchange=ikev2
       dpdaction=clear
       dpddelay=300s
       rekey=no
       left=%any
       leftsubnet=0.0.0.0/0,::/0
       leftauth=pubkey
       leftsendcert=always
       leftcert=server-cert.pem
       leftid=your-vpn-domain.com
       right=%any
       rightauth=eap-mschapv2
       eap_identity=%any
       rightsourceip=10.10.10.0/24,fd00:10:10:10::/64
       rightdns=8.8.8.8,2001:4860:4860::8888
       auto=add
   ```
   *Замените `your-vpn-domain.com` на реальный домен вашей ноды.*

4. Отключите автоматическую проставку маршрутов StrongSwan в системную таблицу Linux. Это критически важно, так как за перехват трафика будет отвечать виртуальный faked-TUN мост панели:
   ```bash
   echo "install_routes = no" >> /etc/strongswan.d/charon.conf
   ```

### Шаг 1.4. Настройка Policy Based Routing и Маскарадинга пакетов
1. Включите форвардинг входящего трафика (IPv4 и IPv6) на уровне ядра Linux хоста:
   ```bash
   sysctl -w net.ipv4.ip_forward=1
   sysctl -w net.ipv6.conf.all.forwarding=1
   echo -e "net.ipv4.ip_forward=1\nnet.ipv6.conf.all.forwarding=1" >> /etc/sysctl.conf
   ```
2. Создайте скрипт автозапуска правил маршрутизации через файл `/etc/rc.local`:
   ```bash
   nano /etc/rc.local
   ```
3. Вставьте в файл следующую конфигурацию (строго перед строкой `exit 0`):
   ```text
   #!/bin/sh -e
   
   # Заворачивание трафика виртуального пула StrongSwan в интерфейс панели strong-tun
   ip rule add from 10.10.10.0/24 lookup 200 2>/dev/null || true
   ip route add default dev strong-tun table 200 2>/dev/null || true
   
   ip -6 rule add from fd00:10:10:10::/64 lookup 200 2>/dev/null || true
   ip -6 route add default dev strong-tun table 200 2>/dev/null || true
   
   # Включение NAT-маскарадинга для мобильного IPv6 трафика
   ip6tables -t nat -A POSTROUTING -s fd00:10:10:10::/64 -j MASQUERADE
   
   exit 0
   ```
4. Выдайте файлу права на исполнение и перезапустите сетевые службы:
   ```bash
   chmod +x /etc/rc.local
   systemctl restart strongswan-starter
   ```

### Шаг 1.5. Конфигурация панели 3x-ui для интеграции со StrongSwan

Для того чтобы трафик от демона StrongSwan корректно перехватывался ядром Xray и проходил сквозь GeoIP-маршрутизацию, необходимо создать транзитный входящий узел (Inbound) на базе протокола `tun`.

Ниже представлены точные параметры конфигурации, распределенные по вкладкам графического интерфейса панели `3x-ui`.

#### 🖼 Вкладка 1: «Основное»
При добавлении нового входящего подключения заполните поля в первой вкладке:

*   **Включить:** `Активировано` (синий тумблер).
*   **Примечание:** `Ipsec` (произвольное текстовое имя узла).
*   **Протокол:** `tun` (выберите из выпадающего списка).
*   **Адрес:** `10.10.10.0/24, fd00:10:10:10::/64` (IP-пулы виртуальной сети StrongSwan для IPv4 и IPv6, которые мы прописали в директиве `rightsourceip` файла `/etc/ipsec.conf`).
*   **Стратегия адреса для ссылок:** `Адрес прослушивания inbound`.
*   **Порядок в подписке:** `3`
*   **Порт:** `13229` (локальный порт, на который ядро операционной системы будет пересылать пакеты).
*   **Общий расход:** `0`
*   **Сброс трафика:** `Никогда`
*   **Дата окончания:** Оставьте поле пустым.

#### 🖼 Вкладка 2: «Протокол»
Перейдите во вторую вкладку верхней навигации и заполните специфичные параметры TUN-интерфейса ядра Xray:

*   **Имя интерфейса:** `strong-tun` (системное имя виртуальной сетевой карты, которое Xray поднимет в ОС хоста).
*   **MTU:** `1500` (стандартный размер максимального пакета для исключения фрагментации трафика).
*   **Gateway:** не трогаем
*   **DNS:** не трогаем
*   **Уровень пользователя:** `0`
*   **Авто-маршруты системы:** не трогаем
*   **Авто-интерфейс исходящих:** `auto`

#### 🖼 Вкладка 3: «Сниффинг»
Перейдите в третью вкладку для настройки глубокого анализа пакетов (инспекции трафика), чтобы ядро Xray могло корректно распознавать заблокированные домены и применять к ним GeoIP-правила маршрутизации:

*   **Включить:** `Активировано` (синий тумблер).
*   **Типы трафика (Теги):** Выберите из выпадающего списка четыре обязательных значения: `HTTP`, `TLS`, `QUIC`, `FAKEDNS`.
*   **Только метаданные:** `Деактивировано` (серый тумблер).
*   **Только маршрутизация:** `Деактивировано` (серый тумблер).
*   **Исключённые IP:** Оставьте поле пустым.
*   **Исключённые домены:** Оставьте поле пустым.

После заполнения всех трех вкладок нажмите синюю кнопку **«Сохранить изменения»** в правом нижнем углу

## 🤖 ЧАСТЬ 2. РАЗВЕРТЫВАНИЕ УПРАВЛЯЮЩЕГО TELEGRAM-БОТА

## 💳 ЧАСТЬ 3. ПОДКЛЮЧЕНИЕ ПЛАТЕЖНОЙ СИСТЕМЫ CRYPTO BOT

Интеграция с Crypto Pay позволяет полностью автоматизировать прием платежей в криптовалютах (USDT, TON, BTC, ETH, NOT и др.) без необходимости развертывания собственных блокчейн-нод.

### Шаг 3.1. Получение API-токена в Telegram
1. Откройте официального бота платежной системы: [@CryptoBot](t.me/send?start=r-fkj43).
2. Перейдите в главное меню и выберите раздел **Crypto Pay** -> **Мои приложения** (My Apps).
3. Нажмите кнопку **Создать приложение** (Create App).
4. Переключите режим работы приложения в зависимости от ваших текущих целей:
   * **Для тестирования:** Создайте приложение в боте [@CryptoTestnetBot](https://t.me/CryptoTestnetBot?start=r-27pww). Скопируйте полученный API-токен и укажите в `.env` флаг `IS_NET_TEST=true`.
   * **Для живых продаж (Production):** Создайте приложение в основном `@CryptoBot`. Скопируйте боевой API-токен и укажите в `.env` флаг `IS_NET_TEST=false`.

### Шаг 3.2. Настройка Webhooks (Опционально)
Платформа по умолчанию использует асинхронный метод ручной проверки статуса счетов через кнопку `🔄 Проверить оплату` с защитой от Race Condition на уровне FSM-контекстов приложения. Это гарантирует 100% срабатывание триггеров активации даже без настройки внешних вебхуков и SSL-сертификатов на стороне сервера бота. 

Если вы планируете включить автоматическую активацию по факту оплаты (Webhooks):
1. В настройках вашего приложения внутри `@CryptoBot` нажмите кнопку **Webhooks**.
2. Введите URL-адрес вашего бота, на который платежная система будет отправлять POST-запросы: `https://your-bot-domain.com`.
3. Нажмите **Включить Webhooks** (Enable Webhooks).

Все поступающие средства от продажи подписок мгновенно зачисляются на внутренний баланс вашего приложения внутри `@CryptoBot`.
